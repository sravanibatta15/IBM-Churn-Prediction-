About the Machine Learning Model

This Customer Churn Prediction system uses a Gradient Boosting Classifier trained on a telecom customer dataset to identify customers likely to discontinue their service.

Model Workflow

  • Data preprocessing with missing value handling and standardization using StandardScaler.
  • Categorical features encoded using One-Hot Encoding.
  • Numeric columns like MonthlyCharges and TotalCharges are normalized for model stability.
  • The model predicts binary output: 0 → Customer stays, 1 → Customer churns.

Features Used

  • Demographics: Gender, SeniorCitizen, Partner, Dependents
  • Services: PhoneService, InternetService, OnlineBackup, TechSupport, etc.
  • Billing & Contract: PaymentMethod, Contract, PaperlessBilling
  • Numerical: MonthlyCharges, TotalCharges

Purpose

The model assists telecom companies in retaining customers by identifying high-risk customers and enabling proactive retention strategies.
